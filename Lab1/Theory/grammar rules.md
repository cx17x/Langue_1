## Построение дерева и обработка синтаксическо модели
За основу берется определенный вариант, описанный в файле text.txt. Ниже описаны правила по которым будет восприниматься синтаксическая модель:
### 1
```source: sourceItem*;``` — «программа (source) — это ноль или больше (*) элементов sourceItem».
Проще: файл может содержать много сущностей верхнего уровня.

### 2

```typeRef: { builtin | custom | array };```
— «ссылка на тип» может быть:
builtin — встроенный тип: bool|byte|int|uint|long|ulong|char|string;
custom — пользовательский тип: любой identifier;
array — массив вида array [ , , ] of typeRef (число запятых задаёт размерность; потом — базовый тип массива, который сам может быть любым typeRef, т.е. рекурсивно).
